Welcome to Space Escape!

The objective of this game is to survive as long as you can while getting the highest score you can. If you get hit by an enemy ship, your run is over.
You can gain points by shooting at and destroying oncoming enemy ships, and also just surviving.

There are 10 levels in this game, starting at level 1. The levels will increase over time, and as they do, so do the enemy ships' speed and health. Destroying enemy ships and surviving, however, will also give you more points.

To help you out, powerups will appear on screen periodically. When you pick one up, your firing mode is changed to that of that powerup, and you can view ammo remaining for that firing mode at the top of the screen.

CONTROLS

Use the arrow keys to move, spacebar to shoot, and the left ctrl key to toggle between firing modes when available (selected mode marked in yellow).

Five requirements have been used in this program:

1. DOM element creation, deletion or modification - bullets, enemies, and the player are the types of elements that are created, modified, and removed throughout the program.

2. DOM traversal - though I am not sure if it counts, document.getElementsByClass() is a commonly used method to go through and retrieve the DOM elements I need to work with, examples being use in the tracking gun's ability to dynamically find targets and menu status selection with the yellow highlight.

3. Capturing and handling events - this should go without saying. From ship movement to toggling firing modes to shooting bullets, this game sure uses event handling.

4. Creating and handling a data structure (JSON, custom objects, etc) - the data for each enemy in this game is created using a custom "Enemy" constructor object. JSON is also used to write to and retain a high score in local storage.

5. Closures - while slightly dissimilar from what we learned from the lesson, scope is handled through the use of the method setTimeout. As an example, the function playerShootController is a function that has a few nested functions to create and add behavior to new bullet elements. The runtime on this playerShootController function is very short, however the scope needs to be retained for animation to work. The bulletTracker function uses setTimeout to keep it running as long as needed independent of the scope it was created in while using the variables passed by the outer function scope it was created in to keep the bullet animation working. This function will run until certain conditions are met, such as hitting an enemy or reaching the end of the screen, where the element the function is working on gets removed from the parent.

Function was favored over optimization. The amount of recursion causes some noticable lag.
